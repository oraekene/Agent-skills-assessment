# Phase 3.5 Results

Phase 3.5 defines the first real internal results surface.

## Files

- `phase-3.5-internal-results-surface.md`
  - rules for grouped bands, shortlists, detail views, and better alternatives
- `internal-results-surface.schema.json`
  - machine-readable results-surface schema
- `phase-3.5-results-surface-example.round1.json`
  - first machine-readable example results surface
- `phase-3.5-worked-simulation.md`
  - worked examples of domain browse, shortlist, detail, and better alternatives

## What Is Now Available

The internal engine can now specify how to present:

- grouped-band domain results
- task shortlists
- per-skill detail surfaces
- better-alternative suggestions
- no-match states

## Practical Outcome

The project now has a clear bridge from:

- machine-readable ranking data

to:

- human-usable internal result views

## Main Limitation

Phase 3.5 is still a specification and example layer.

It does not yet implement:

- a live UI
- interactive expansion state
- persisted per-user result preferences
- real-time runtime-overlay merging
